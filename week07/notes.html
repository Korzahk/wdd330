Notes

  • error - something went wrong
  • user errors are really programmer errors
  • exception - error that returns a value the program can use to deal with the error
  • stack trace - call sequence that leads to the error
  • strict mode makes styling errors real errors
    put 'use strict'; at the top of the body of a function to use
  • linting tools are another option
  • window.navigator.userAgent identifies the user's browser
  • javascript has lots of error objects
  • jest is a library of userful testing tools
